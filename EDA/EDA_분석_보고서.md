## 2. 폴더 구조

```
train_annotations/
└── K-001900-016548-019607-029451_json/   ← 이미지에 포함된 약 조합
    ├── K-001900/                          ← 약 고유 ID
    │   ├── *_70.json                      ← 촬영 각도 70도
    │   ├── *_75.json                      ← 촬영 각도 75도
    │   └── *_90.json                      ← 촬영 각도 90도
    ├── K-016548/
    ├── K-019607/
    └── K-029451/
```

- 폴더명의 6자리 숫자 = 약의 고유 번호
- 하나의 이미지에 최대 4개의 약이 함께 촬영됨
- 세트 폴더 총 114개 (3종 조합 89세트 / 4종 조합 25세트)

## 8. 클래스 불균형 분석

| 항목 | 수치 |
|------|------|
| 전체 클래스 수 | 56개 |
| 소수 클래스 수 | 17개 (하위 20%, 어노테이션 3개 이하) |
| 최대 불균형 비율 | **51배** |

### 대응 방법 비교

| 방법 | 설명 | 장점 | 단점 |
|------|------|------|------|
| 데이터 증강 | 회전/flip/밝기 조정으로 늘리기 | 데이터 다양성 확보 | 원본과 유사한 이미지만 생성 |
| 클래스 가중치 | 소수 클래스에 높은 loss 가중치 | 구현 간단 | 다수 클래스 성능 저하 가능 |
| 오버샘플링 | 소수 클래스 이미지 복제 | 빠른 적용 | 과적합 위험 |
| 언더샘플링 | 일양하이트린정 153개 → 줄이기 | 균형 확보 | 데이터 손실 |
| **WeightedRandomSampler** | DataLoader에서 소수 클래스 더 자주 샘플링 | 데이터 손실 없음 | 학습 속도 저하 가능 |

> ✅ **권장**: WeightedRandomSampler + 소수 클래스 데이터 증강 병행

## 9. 전처리 전략 요약

| # | 문제 | 원인 | 대응 전략 |
|---|------|------|-----------|
| 1 | 환경 편향 | 배경 100% 연회색, 조명 100% 주백색 | Color Jitter, 배경 합성, 노이즈/블러 |
| 2 | 반쪽 알약 부재 | 분할선 있는 약은 많지만 실제 반쪽 약 이미지 없음 | CoarseDropout |
| 3 | 각인 선명도 차이 | 이미지마다 해상도·각도 차이 | Sharpening, 1024px 리사이즈 |
| 4 | 클래스 불균형 | 최대 51배 차이 | WeightedRandomSampler, 추가 수집 |
| 5 | 뒷면 데이터 부족 | 앞면 98.4%, 뒷면 1.6% | 뒷면 데이터 추가 수집 |

### Albumentations 증강 파이프라인 요약

```python
train_transform = A.Compose([
    A.LongestMaxSize(max_size=1024),          # ① 고해상도 리사이즈
    A.PadIfNeeded(1024, 1024),                # ① 패딩
    A.ColorJitter(..., p=0.7),                # ② 조명/색상 변조
    A.OneOf([GaussNoise, MotionBlur, GaussianBlur], p=0.4),  # ③ 노이즈/블러
    A.CoarseDropout(..., p=0.4),              # ④ 반쪽 알약 대응
    A.HorizontalFlip(p=0.5),                  # ⑤ 기하학적 변환
    A.VerticalFlip(p=0.3),
    A.Rotate(limit=30, p=0.5),
    A.Sharpen(..., p=0.3),                    # ⑥ 각인 선명화
    A.Normalize(...),                          # ⑦ 정규화
    ToTensorV2(),
], bbox_params=BboxParams(format='coco', min_visibility=0.3))
```

## 10. 모델링 팀 인계 사항

| 항목 | 권장값 | 근거 |
|------|--------|------|
| 입력 해상도 | 1024 × 1024 | mAP@0.75+ 기준, 작은 객체 디테일 보존 |
